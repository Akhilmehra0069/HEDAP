Methods

  • clearAddrOverride
    clear contacts' Address Override verify: contacts' Mailing address fields updated to the HH default address
  • createHHTestData
    utility to set up common test data for the Address tests. - this will create & insert N household Accounts, each with M contacts - also will create a Default Addresses for each HH Account - will also setup the default Account Processor to be HH Accounts
  • initTestAddr
    Initializes (but not inserts) N new Address objects.
  • insertContactWithAddress
    insert contact w/ address verify: new default Address created contact address lookup updated to new address the contact gets the new Address
  • insertContactWithAddressOverride
    insert contact w/ new address override verify: new non-default Address created contact address lookup updated to new address the contact gets the new Address
  • insertContactWithMultilineStreetAddress
    insert contact w/ multiline street address verify: new default Address created and the street address is broken up into street and street2 the HH Account gets the new multiline address
  • insertConWithAddrToExistingHH
    insert new contacts with addresses to existing households that have default addresses verify: all contacts and hh get the new default hh address from the new contacts
  • insertConWithDefaultAddrToExistingHH
    insert new contacts with default addresses to existing households that have default addresses (this is to mimic the behavior of doing New Contact from the HH Account, in which case Salesforce brings up the new contact page in edit mode, with the account's billing address copied into the contact's mailing address fields). verify: no new addresses created new contacts point at the default address
  • insertConWithNoAddrToExistingHH
    insert new contacts with no addresses to existing households that have default addresses verify: new contacts get the default hh address
  • insertNewDefaultAddr
    insert new default addresses to HH w/ existing default addresses verify: contact's && hh address matches new default old default addresses no longer marked default
  • insertNewDefaultAddrAndReset
    insert new default addresses to HH w/ existing default addresses, and then go back verify: contact's && hh address matches original default new default addresses no longer marked default
  • insertNewDefaultMultilineStreetAddr
    insert new default multiline street addresses to HH w/ existing default addresses verify: contact's && hh address matches new default street & street2 combined into contact and account single street field
  • insertNewSeasonalAddr
    insert new seasonal addresses to HH w/ existing default addresses verify: contact's && hh address matches new seasonal old default addresses still marked default
  • insertNewSeasonalAddrNotCurrent
    insert new seasonal addresses, not current, to HH w/ existing default addresses verify: contact's && hh address stay with default old default addresses still marked default
  • insertPartialSeasonalAddr
    create multiple partial seasonal address verify: partial seasonal detected, seasonal addresses don't save
  • insertSeasonalAddrOverlapExisting
    create multiple seasonal address that overlap existing seasonal addresses verify: overlap detected, seasonal addresses don't save
  • insertSeasonalAddrWithOverlap
    create multiple seasonal address that overlap with eachother verify: overlap detected, seasonal addresses don't save
  • isMatchAddressAccCon
    Returns whether an Account Billing address matches a Contact Mailing address
  • newAddrForOrgAccounts
    update N organizational accounts with address objects verify: account addresses updated contact addresses not updated
  • newAddrNoAccount
    create an Address with no Account verify: runtime error
  • newDefaultAddr
    create N new default Addresses for N Households (with M Contacts) verify: N addresses created N household billing address fields updated M contact Mailing address fields updated
  • newDupeAccountAddr
    create N new duplicate Addresses from Account verify: duplicates not created
  • newDupeAccountAddrNonDefault
    create N new duplicate Addresses from Account, with non-default existing verify: duplicates not created found match becomes Default
  • newDupeContactAddr
    create N new duplicate Addresses from Contact verify: duplicates not created
  • newDupeContactAddrNonDefault
    create N new duplicate Addresses from Contact, with non-default existing verify: duplicates not created found match becomes Default
  • newOrgAccounts
    create N new organizational accounts with billing addresses verify: N addresses created account address matches address object address
  • scheduleSeasonalAddrUpdate
    schedule a seasonal address update verify: contact's && hh address matches new seasonal old default addresses still marked default
  • setConAddrNoOverride
    set contact current address, but not override verify: contact's address updated, but not override
  • setConAddrOverride
    set contact address override verify: contact's address matches override
  • testDisabledHHAccountAddr
    create N contacts with Household Accounts when the setting for Address Mgmt is off. verify: no Address objects created HH Account addresses set Contact addresses set
  • testSeasonalAddrOverlapLogic
    create multiple seasonal address that overlap with eachother verify: overlap detected
  • updateAccAddrExisting
    update accounts' billing address to cause an update of the default Address object verify: existing Default Address updated HH and Contacts get updated Address
  • updateAccAddrNew
    update accounts' billing address to cause creation of new default Address object verify: new Default Address created HH and Contacts get new Address
  • updateAccMultilineStreetAddr
    update accounts' billing address with multiline street address verify: new Default Address created HH and Contacts get new Address new Addresss has multiline street split into Street and Street2 fields
  • updateAddrOverride
    update contacts' Address Override verify: contacts' Mailing address fields updated to the Address Override's fields.
  • updateConAddrExisting
    update contacts' mailing address to cause an update to default Address object verify: existing Default Address updated HH and Contacts get updated Address
  • updateConAddrExistingCaseOnly
    update contacts' mailing address with just a case change, to cause an update to default Address object verify: existing Default Address updated HH and Contacts get updated Address
  • updateConAddrMarkOverride
    update contacts' mailing address and mark it as override, to cause creation of new non-default Address object verify: new Address created HH sticks with default Address contact gets new address
  • updateConAddrNew
    update contacts' mailing address to cause creation of new default Address object verify: new Default Address created HH and Contacts get new Address
  • updateConAddrWithOverride
    update contacts' mailing address when they have an address override verify: new non-default Address created contact override updated to new address only the contact gets the new Address
  • updateDefaultAddr
    update existing default Addresses verify: household billing address fields updated contact Mailing address fields updated
  • updateNonDefaultAddr
    update existing non-default Addresses verify: no change to HH and contact addresses
  • updateOrgAccounts
    update N organizational accounts with billing addresses verify: N addresses created account addresses updated contact addresses not updated
  • updateSeasonalAddr
    update seasonal addresses to current, on HH w/ existing default addresses verify: contact's && hh address updated to seasonal address old default addresses still marked default
